# 10. 需求分析

本文介绍如何在 Vibe Coding 时代发现和验证产品需求。

## 什么是需求分析？

需求分析是确定产品要解决什么问题、满足什么需求的过程。

## Vibe Coding 时代的优势

过去：
- 需要技术背景才能验证想法
- 开发原型成本高
- 验证周期长

现在（Vibe Coding）：
- 几分钟就能做出原型
- 快速验证市场需求
- 降低试错成本

## 需求发现方法

### 1. 解决自己的问题

```
问自己：
- 日常工作中有什么重复性任务？
- 有什么工具可以更好？
- 有什么痛点一直想解决？
```

### 2. 观察他人

```
- 朋友、家人的抱怨
- 社交媒体上的求助
- 论坛里的常见问题
```

### 3. 竞品分析

```
- 使用现有产品
- 记录不方便的地方
- 思考如何改进
```

## 需求验证

### 最小可行产品 (MVP)

不要一开始就做完整的产品：

```
MVP = 核心功能 + 最小实现
```

### 快速验证流程

```
1. 想法 → 2. 原型 → 3. 测试 → 4. 迭代
```

## 需求文档模板

```markdown
# 需求文档

## 问题
- 用户面临什么问题？
- 这个问题多严重？

## 解决方案
- 我们的产品如何解决这个问题？
- 核心功能是什么？

## 目标用户
- 谁会用这个产品？
- 他们有什么特点？

## 成功指标
- 如何衡量产品成功？
- 预期达到什么效果？
```

## 案例：亲戚计算器

**需求发现**：逢年过节被问"这个亲戚怎么叫"

**解决方案**：输入关系，自动计算称呼

**验证方式**：朋友圈分享，收集反馈
