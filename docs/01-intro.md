# 01. Vibe Coding 入门指南

什么是 Vibe Coding？为什么它正在改变编程？

## 传统编程 vs Vibe Coding

| 方面 | 传统编程 | Vibe Coding |
|------|---------|-------------|
| 输入 | 代码 | 需求描述 |
| 方式 | 敲代码 | 对话+自然语言 |
| 重点 | 实现细节 | 最终效果 |
| 调试 | 逐行排查 | AI 协助分析 |
| 效率 | 小时/天 | 分钟 |

## 核心理念

> 用 AI 编程，而不是被编程

1. **描述需求** → AI 生成代码
2. **自然语言** → 替代命令行
3. **AI 协助** → 调试不再孤单
4. **快速迭代** → 从想法到 Demo 只要几分钟

## 谁适合 Vibe Coding？

✅ **创业者** — 快速验证想法
✅ **产品经理** — 快速原型
✅ **设计师** — 实现交互原型
✅ **初学者** — 学习编程思维
✅ **全栈工程师** — 10x 开发效率

## 快速开始

### 第一步：选择工具

建议初学者从 **OpenClaw** 开始：

```bash
# 安装
npm install -g openclaw

# 初始化
openclaw init

# 启动
openclaw start
```

### 第二步：下第一个指令

```
"帮我创建一个计算器 Web 应用"
```

### 第三步：迭代优化

```
"把按钮改成蓝色"
"添加历史记录功能"
"改成暗黑模式"
```

## 工作流程

```
1. 描述需求 (What)
       ↓
2. AI 生成代码 (Generate)
       ↓
3. 测试体验 (Test)
       ↓
4. 自然语言修改 (Refine)
       ↓
5. 重复 3-4 直至完成
```

## 常见误区

❌ **不要** 试图一次描述完整需求  
✅ **应该** 迭代式描述，小步前进

❌ **不要** 完全依赖 AI，不理解代码  
✅ **应该** AI 生成 + 人工理解 + 优化

❌ **不要** 忽视代码审查  
✅ **应该** 定期让 AI 审查生成的代码
