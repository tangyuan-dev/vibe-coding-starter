# 07. 对话工程

本文介绍如何与 AI 进行有效对话，获得更好的结果。

## 什么是对话工程？

对话工程（Conversation Engineering）是关于如何向 AI 提问、如何管理对话上下文的学问。

## 核心原则

### 1. 明确具体

❌ 不好：
```
帮我写个代码
```

✅ 好的：
```
用 Python 写一个 HTTP 服务器，监听 8080 端口，返回 JSON 格式的当前时间
```

### 2. 提供上下文

❌ 不好：
```
这个 bug 怎么修？
```

✅ 好的：
```
在 Node.js Express 项目中，登录接口返回 500 错误，错误信息是 "Cannot read property 'id' of undefined"，这是用户登录模块的代码...
```

### 3. 指定输出格式

❌ 不好：
```
给我一个 README
```

✅ 好的：
```
生成一个开源项目的 README，包含：
1. 项目标题（用 # ）
2. 简短描述（1-2 行）
3. 安装步骤（用代码块）
4. 使用示例（用代码块）
5. License
```

## 提示词模板

### 代码生成

```markdown
我想用 [技术栈] 做一个 [功能描述]。

要求：
- [需求1]
- [需求2]

请生成完整的代码，如果需要多个文件请列出文件结构。
```

### Bug 修复

```markdown
遇到一个 bug：
[错误信息/错误截图]

复现步骤：
1. [步骤1]
2. [步骤2]

相关代码：
[代码片段]

帮我：
1. 分析可能原因
2. 提供修复方案
3. 解释原理
```

### 代码审查

```markdown
请审查这段代码：
[代码]

关注点：
- 安全性
- 性能
- 可维护性
- 最佳实践

请给出具体改进建议。
```

## 进阶技巧

### 1. 分步提问

```markdown
第一步：帮我设计一个用户管理模块的数据库结构

第二步：基于这个结构，生成 Node.js 的 Sequelize 模型

第三步：生成对应的 RESTful API
```

### 2. 迭代优化

```markdown
第一版：生成一个基本的待办事项应用

反馈：功能太简单，添加拖拽排序和过期提醒

反馈：改成暗黑模式，并支持数据同步
```

### 3. 指定角色

```markdown
你是一个资深 Node.js 工程师，有 10 年经验。
请帮我审查这段代码...
```

### 4. 提供示例

```markdown
生成一个 API 文档，类似这样的格式：
```json
{
  "GET /users": {
    "description": "获取用户列表",
    "params": { "page": "页码" }
  }
}
```
"""

## 常见问题

### Q: AI 回答太简略怎么办？

A: 添加"请详细解释"或"请分步骤说明"

### Q: AI 理解错了我的意思怎么办？

A: 重新描述需求，并指出具体哪部分理解错了

### Q: 代码有 bug 怎么办？

A: 直接把错误信息发给 AI，让它修复
