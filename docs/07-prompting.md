# 07. 对话工程 | Prompt Engineering

让 AI 理解你的意图，获得更好结果的艺术。

## 🎯 为什么对话重要？

Vibe Coding 的核心是 **与 AI 对话**，而不是写代码。

| 对话质量 | AI 输出 | 效率 |
|----------|---------|------|
| 模糊笼统 | 需要多次修改 | ⏳ 慢 |
| 清晰具体 | 一次通过 | ⚡ 快 |

## 📝 基础原则

### 1. 明确任务

❌ 错误：
```
"帮我写个东西"
```

✅ 正确：
```
"帮我写一个 Python 脚本，读取 CSV 文件，
计算每列的平均值并输出到新文件"
```

### 2. 提供上下文

❌ 错误：
```
"修复这个 bug"
```

✅ 正确：
```
"修复 index.js 中的登录 bug。
错误现象：用户输入正确密码后返回 'invalid password'
预期行为：密码正确应该登录成功
已尝试：检查了数据库连接正常"
```

### 3. 指定格式

❌ 错误：
```
"给个代码"
```

✅ 正确：
```
"用 Python 写一个接口，返回 JSON 格式：
{
  "code": 0,
  "message": "success",
  "data": {...}
}"
```

## 🛠️ 高级技巧

### 指定角色

```
你是一个 10 年经验的 Python 后端工程师。
精通 Django、Flask，有丰富的 API 设计经验。

请帮我评审这段代码...
```

### 分步指令

```
第一步：分析需求
第二步：设计数据库结构  
第三步：实现 API 接口
第四步：编写单元测试
```

### 示例驱动

```
按照以下示例的格式和风格写代码：

示例：
```python
def get_user(user_id):
    user = db.query("SELECT * FROM users WHERE id = ?", user_id)
    return UserSchema().dump(user)
```

请写一个获取文章列表的函数...
```

### 约束条件

```
写一个排序算法，要求：
1. 时间复杂度 O(n log n)
2. 空间复杂度 O(n)
3. 使用 Python 实现
4. 添加类型注解
5. 包含单元测试
```

## 🎨 场景模板

### 写代码

```
作为 [语言] 开发者，写 [功能] 代码。

要求：
- 符合 [代码规范]
- 包含 [测试]
- 注释清晰

[具体需求描述]
```

### 代码审查

```
作为资深开发者，审查以下代码：

[代码]

请检查：
1. 潜在 bug
2. 性能问题
3. 安全漏洞
4. 代码规范
```

### 解释代码

```
解释以下代码的功能和原理：

[代码]

用通俗易懂的语言解释，适合 [初学者/有经验者] 理解
```

### 重构优化

```
重构以下代码，提升 [性能/可读性/可维护性]：

[代码]

改进后请说明：
1. 做了哪些改动
2. 为什么这样改
3. 改进了什么
```

## ⚠️ 常见问题

### 1. 过度简化

❌ "写个登录"

✅ "写一个用户登录功能，包含：
- 用户名密码验证
- 错误提示
- 记住登录状态
- 防止暴力破解（5次错误后锁定5分钟）"

### 2. 缺少信息

❌ "API 报错，帮我看看"

✅ "API 返回 500 错误：
- 请求路径：/api/user
- 请求方法：POST
- 请求体：{name: 'test'}
- 错误信息：...（附上完整错误堆栈）
- 环境：Node.js + Express
- 已尝试：...（附上你的调试过程）"

### 3. 矛盾指令

❌ "代码要简单，但功能要完整，还要高性能"

✅ "优先级：性能 > 简单 > 功能完整"

## 🔧 OpenClaw 对话技巧

### 使用系统提示

在 OpenClaw 中可以设置系统提示：

```yaml
# ~/.openclaw/config.yaml
system_prompt: |
  你是一个专业的 Python 开发者。
  代码风格遵循 PEP 8。
  总是先思考再回答。
```

### 分割大任务

```
先创建数据库模型
再写 API 接口
然后写前端页面
最后写测试
```

### 迭代优化

```
第一轮：生成基础代码
第二轮：添加错误处理
第三轮：优化性能
第四轮：添加文档
```

## 📊 对话质量评估

| 指标 | 差 | 一般 | 好 |
|------|-----|------|-----|
| 清晰度 | 模糊 | 基本清楚 | 明确具体 |
| 完整性 | 缺这缺那 | 基本完整 | 面面俱到 |
| 可执行性 | 不知道要什么 | 知道要什么 | 可直接执行 |
| 效率 | 需要多轮 | 1-2 轮 | 一次搞定 |

## 🧪 练习

试试用这些不同的方式提问，比较 AI 的回复质量：

```python
# 基础
"写个排序"

# 中级
"用 Python 实现快速排序算法，需要完整可运行的代码"

# 高级
"""
作为 Python 工程师，实现快速排序算法。
要求：
1. 时间复杂度 O(n log n)
2. 包含类型注解
3. 添加单元测试
4. 代码符合 PEP 8
"""
```
