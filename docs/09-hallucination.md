# 09. 幻觉处理

本文介绍如何识别和处理 AI 幻觉。

## 什么是 AI 幻觉？

AI 幻觉是指 AI 生成看似正确但实际错误的内容，包括：
- 虚构的代码或 API
- 错误的文档描述
- 不存在的功能

## 如何识别幻觉？

### 1. 代码无法运行

```
AI: 使用 fs.readFileSync() 读取文件
实际: Node.js 应该用 fs.promises.readFile() 或 require('fs').readFileSync
```

### 2. 文档与实际不符

```
AI: 这个库的 API 是 xxx
实际: 官方文档显示 API 是 yyy
```

### 3. 过度自信的语气

```
AI: "这个方法绝对有效" → 需要警惕
AI: "可能需要检查..." → 更有可能是真的
```

## 处理方法

### 1. 验证代码

```bash
# 运行代码，检查是否报错
npm run dev

# 检查依赖是否正确安装
npm list [package-name]
```

### 2. 查阅官方文档

```
AI 说的 API 对不对？
→ 打开官方文档验证
→ 搜索 GitHub issues
```

### 3. 分步验证

```markdown
第一步：先让 AI 生成最简单的版本

第二步：运行测试，确认基础功能

第三步：再逐步添加复杂功能
```

### 4. 明确指出不确定

```markdown
你不确定的地方请说"需要查证"，
不要猜测。
```

## 预防技巧

### 1. 提供真实上下文

```markdown
这是项目的 package.json：
{
  "dependencies": {
    "express": "^4.18.0"
  }
}

不要使用版本不兼容的 API
```

### 2. 指定具体版本

```markdown
使用 Express 4.x 版本的 API
不要使用 Express 5.x 的新特性
```

### 3. 要求提供参考

```markdown
生成代码时，请同时提供：
1. 官方文档链接
2. 关键代码的解释
```

## 常见幻觉类型

| 类型 | 示例 | 解决方法 |
|------|------|---------|
| 虚构 API | "xxx库有 yyy 方法" | 查文档 |
| 错误语法 | "这样写就可以" | 运行测试 |
| 过时信息 | "这个方法已弃用" | 查最新文档 |
| 过度简化 | "一行代码搞定" | 分步验证 |
