# 04. Web 应用开发

手把手开发一个完整的 Web 应用。

## 项目选择

### 初学者项目
- Todo 待办清单
- 笔记应用
- 天气查询
- 汇率转换器

### 中级项目
- 博客系统
- 任务管理工具
- 简易电商后台
- 聊天室

### 高级项目
- AI 聊天助手
- 数据可视化仪表盘
- 在线代码编辑器
- 小型 SaaS 产品

## 技术栈选择

| 类型 | 选项 | 适用场景 |
|------|------|----------|
| 前端框架 | React / Vue / Svelte | 现代 Web 开发 |
| 后端 | Node.js / Python / Go | API 开发 |
| 数据库 | PostgreSQL / MongoDB / SQLite | 数据存储 |
| 部署 | Vercel / Netlify / Railway | 快速上线 |

## 开发流程

### 1. 需求分析
明确功能列表、用户群体、使用场景

### 2. 技术选型
根据团队熟悉度和项目需求选择技术栈

### 3. 原型设计
使用 Figma 或手绘草图设计界面

### 4. 数据库设计
设计数据表结构、关系模型

### 5. 接口设计
规划 API 端点和数据结构

### 6. 编码实现
按模块逐步开发

### 7. 测试验收
功能测试、用户体验优化

### 8. 部署上线
配置 CI/CD，部署到生产环境

## Vibe Coding 技巧

### 让 AI 更好地理解项目

```
项目是一个 [类型] 应用，主要功能包括：
1. [功能 1]
2. [功能 2]
3. [功能 3]

技术栈：
- 前端：[框架]
- 后端：[框架]
- 数据库：[数据库]

请先设计数据库结构，然后逐步实现功能。
```

### 分步开发

```
第一步：创建项目结构和基础配置文件
第二步：实现数据库模型和 API
第三步：开发前端页面和交互
第四步：添加样式和响应式设计
第五步：测试和修复问题
```

### 遇到问题

```
遇到了 [具体错误/问题]，错误信息是：
[粘贴错误]

我尝试的解决方案：
1. [尝试 1]
2. [尝试 2]

请帮我分析原因并提供修复方案。
```

## 实战示例：Todo 应用

### 功能需求
- 添加任务
- 标记完成
- 删除任务
- 筛选（全部/进行中/已完成）

### AI 对话示例

```
帮我创建一个 Todo 应用，使用 React + Vite + localStorage。
功能：
1. 输入任务名称添加
2. 点击切换完成状态
3. 删除按钮
4. 底部显示统计（进行中/已完成）

样式：简洁风格，绿色主色调
```
